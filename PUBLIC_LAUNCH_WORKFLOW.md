# F1 车模情报站公开试运营 Workflow

目标：先把它做成一个可信、轻维护、能给小红书账号引流的免费情报站，再根据真实玩家反馈决定要不要商业化。

## 1. 本地产品闭环

- [x] 抓 Stone Model Car 和 GPworld。
- [x] 抓取图片、来源链接、标题、摘要、比例、车队、车手。
- [x] 后台审核后再发布，避免错误内容直接上首页。
- [x] 首页按日期排序，支持搜索和筛选。
- [x] RSS、sitemap、robots。
- [x] 后台登录 cookie，不再用 URL token。
- [x] 抓取锁，避免重复抓取撞车。
- [x] 今日新增、近 7 天新增。
- [x] 原始来源链接和分享用 OpenGraph metadata。

## 2. 公开上线前

- [x] 本地设置长随机 `F1_RADAR_ADMIN_TOKEN`，保存在 `.env`。
- [ ] 线上部署时在平台环境变量里重新设置 `F1_RADAR_ADMIN_TOKEN`。
- [ ] HTTPS 部署时设置 `F1_RADAR_COOKIE_SECURE=1`。
- [ ] 单独建一个 Git 仓库，只放 `f1-model-radar`，不要夹带其他文件。
- [ ] 选择域名，建议先用 `modelgridradar.com` 这类中性名字。
- [ ] 部署到 Render、Fly.io、Railway 或 VPS。
- [ ] 部署后打开 `/admin/login`，手动跑一次“抓取最新”。
- [ ] 检查 `/rss.xml`、`/sitemap.xml`、详情页分享预览。

## 3. 首批用户测试

- [ ] 小红书发第一条介绍帖：不用翻墙看 F1 车模海外新品/预售/补货。
- [ ] 评论区收集 5 个真实玩家问题：他们想看哪个品牌、哪个比例、哪个车手。
- [ ] 每天固定一次从后台发布 5-20 条精选，不追求全量。
- [ ] 记录一周：访问量、收藏/评论、用户要求的新来源。

## 4. 下一批功能

- [ ] 按车队/车手生成订阅 RSS。
- [ ] 图片本地缓存，降低外链失效风险。
- [ ] 每个来源显示最近一次成功抓取和错误详情。
- [ ] 加“热门车手/车队”聚合页，方便 SEO。
- [ ] 小红书分享卡片生成器。

## 5. 是否商业化的判断线

先不卖会员。只有满足下面任意一个条件，再考虑收费：

- 连续两周每天都有自然访问。
- 有 10 个以上玩家主动提交来源或问你能不能加提醒。
- 小红书有人明确说“这个站我会每天看”。
- 有店铺/卖家想在站里露出新品或库存。
